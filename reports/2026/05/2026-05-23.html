Friday rotation: RAG pipelines, tool use patterns & agent architecture

3 Tips
1 RAG / Agent Arch

Agentic RAG — Treat Retrieval as a Tool, Not a Prerequisite

The 2026 best practice for RAG is to stop force-feeding context into every prompt. Instead, expose your vector store as a callable tool (e.g., search_docs) and let the agent decide when retrieval is needed. This cuts token costs by 40-60% on queries that don't require document grounding.

# In your agent's tool definition: tools = [ {"name": "search_docs", "description": "Search internal knowledge base. Use ONLY when the user asks about company-specific policies, procedures, or data."}, {"name": "search_web", "description": "Search the public internet for current events or general knowledge."} ]

The key insight: a lightweight router classifies intent before the expensive retrieval pipeline fires. Simple queries go straight to the LLM; complex multi-hop questions trigger the full Agentic RAG loop of Retrieve → Reason → Retrieve again.

2 MCP / Tool Use

Confluent MCP Servers — Give AI Agents Safe Access to Kafka Streams

Confluent released open-source MCP servers and "Agent Skills" that allow AI coding tools (Claude Code, Codex, Copilot) to safely introspect Kafka topics, read schemas, and reason about CDC pipelines without direct cluster credentials. This solves the "how does my AI agent understand my event-driven architecture?" problem.

# Install Confluent MCP server for Claude Code: claude mcp add confluent-kafka \ --transport stdio \ --env CONFLUENT_API_KEY=$KEY \ --env CONFLUENT_API_SECRET=$SECRET

Your agent can now answer questions like "which consumers are lagging on the payments topic?" or "show me the Avro schema for order events" — critical for debugging microservices architectures where Kafka is the nervous system.

3 Eval / Guardrails

Faithfulness Judges — The Self-Check Loop That Kills Hallucinations

Before your RAG agent delivers an answer, route it through a smaller "judge" model that scores the draft for faithfulness to the retrieved documents. If the judge detects unsupported claims, force the agent to backtrack and re-retrieve. This single pattern eliminates 70%+ of production hallucinations.

# Pseudo-code for faithfulness gating: draft = agent.generate(query, context) score = judge_model.score_faithfulness(draft, context) if score < 0.85: context = agent.re_retrieve(query, strategy="decompose") draft = agent.generate(query, context) return draft

Cost: ~$0.002 per check using Gemini Flash or Haiku as the judge. The ROI is enormous — one hallucinated answer in production costs more than 10,000 faithfulness checks. Pair with LangSmith or Arize span tracing to monitor judge rejection rates over time.

3 Concepts

The Conductor Pattern — Developer as AI Orchestration Layer

The dominant 2026 developer workflow pattern: a single engineer defines architectures, sets boundaries, and supervises multiple AI agents executing simultaneously — Claude Code for long-running architectural tasks, Cursor for real-time visual edits, Codex for async enterprise jobs. The developer writes almost no code directly but instead "conducts" agents via system prompts, guardrails, and evaluation pipelines.

The practical skill shift: knowing which model to route which task to, how to compose tool-use schemas that constrain agent behavior, and how to build feedback loops that catch agent errors before they compound. Think of it as "engineering management for AI workers" — with the same principles of delegation, verification, and scope control.

ETF Breakeven Ceiling — The $83K Resistance Explained

Bitcoin stalled at $82-83K because that's the average cost basis of US spot-Bitcoin ETF holders. When BTC rallied to breakeven, ETF holders who had been underwater immediately sold to exit at flat — creating a self-reinforcing resistance ceiling. The ninth-largest weekly ETF outflow on record ($1.7B) occurred precisely at this price level.

This is a new structural market dynamic unique to the ETF era: institutional capital creates visible, predictable supply walls at known cost-basis levels. For traders, the implication is clear — breaking above $83K requires enough buying pressure to absorb the entire breakeven selling wave. Once cleared, the next resistance may be significantly higher as the overhang disappears.

Agentic Economy — When AI Agents Need Their Own Financial Rails

NEAR's 50% rally crystallizes a growing thesis: as AI agents become autonomous actors that need to pay for services, receive payments, and manage resources, they need financial infrastructure designed for machines, not humans. Traditional banking (KYC, manual approvals, business hours) doesn't work for agents that transact microsecond-by-microsecond.

Blockchain rails solve this: programmable money, 24/7 settlement, no identity requirements, and smart contracts as enforceable agreements between agents. The market is pricing in a future where AI agents are the primary users of crypto infrastructure — not humans. NEAR, Solana, and Base are all competing to be the "settlement layer for the agentic economy." This narrative explains why AI-adjacent L1s are outperforming pure DeFi tokens in 2026.
