Sunday rotation: hands-on projects combining multiple tools — build something today

3 Tips
1 MLX + SwiftUI

Build an Offline AI Chat App with MLX Swift in One Afternoon

Combine Apple's mlx-swift + MLXLMCommon packages to run a 4-bit quantized Llama-3 8B entirely on your M-series Mac or iPhone. Steps: (1) Add the SPM packages, (2) Download a quantized model via mlx-lm CLI, (3) Use MLXLLM.load() + async token generation in SwiftUI. Zero cloud costs, full privacy, runs offline. Use Cursor to scaffold the SwiftUI views while Claude Code handles the MLX integration layer.

Time: ~3 hours | Tools: Xcode, mlx-swift, Cursor, 4-bit GGUF from HuggingFace

2 MCP + AgentGate

Build a Secure Custom MCP Server with Runtime Authorization

Stand up an MCP server that exposes your local tools (file system, database queries, API calls) and wrap it with AgentGate for runtime trust scoring. Steps: (1) npx @modelcontextprotocol/create-server to scaffold, (2) Register tools with Zod schemas, (3) Add AgentGate middleware for purpose-alignment scoring and human-in-the-loop approvals on writes. (4) Connect to Claude Desktop. Result: your AI agent can read freely but any destructive action requires your approval via Slack webhook.

Time: ~2 hours | Tools: Node.js, MCP SDK, AgentGate, Claude Desktop

3 Dafny + Codex

Your First Vericoding Pipeline: Let AI Write Formally Verified Code

Try the "vericoding" pattern: write a formal spec in Dafny, then have Codex generate both the implementation and the mathematical proof. Steps: (1) Install Dafny extension in VS Code, (2) Write a spec for a simple function (e.g., binary search with bounds guarantee), (3) Prompt Codex: "Implement this Dafny spec and generate the verification proof." (4) Run dafny verify — if it passes, the code is mathematically correct for all inputs. Current LLM success rate on Dafny: 82%.

Time: ~1.5 hours | Tools: VS Code, Dafny, Codex CLI, Z3 solver

Hot Concepts

Structural patterns & paradigm shifts surfacing this weekend

3 Concepts

Vericoding — From Probabilistic Testing to Deterministic Proof

The paradigm shift from "test enough and hope" to "prove once and know." As AI generates 60%+ of code, testing alone can't keep pace with output velocity. Vericoding lets LLMs write formal proofs alongside implementations — Z3 or Lean verify correctness mathematically. Testing shows presence of bugs; vericoding proves their absence.

The economics are compelling: human code review of 1,000 AI-generated lines costs hours; a machine-checked proof costs seconds. The bottleneck shifts from "can AI write correct code?" to "can humans write correct specifications?" — a much smaller, more tractable problem. Expect vericoding to become a standard CI/CD gate for critical paths within 18 months.

Runtime Agent Security — The Death of Static Authorization

OAuth grants static scopes ("this app can read your files"). But AI agents generate actions dynamically — an agent with read access might chain 47 tool calls into an exfiltration pipeline. Runtime security (AgentGate, OWASP Agentic Top 10) treats every agent action as untrusted until scored for purpose-alignment in real-time. The IETF is drafting OAuth extensions with requested_actor parameters for agent identity.

Key mental model shift: security perimeter moves from "who are you?" to "what are you doing right now?" Static RBAC is dead for autonomous systems. Every agentic architecture needs a Policy Decision Point that scores behavioral intent at request time — not just at auth time. This is the "firewall moment" for the agentic era.

The AI Validator Developer — From Writer to Verifier

As AI handles code generation, the developer role bifurcates: "prompt engineers" who direct AI output, and "AI validators" who review, verify, and architect. The validator's edge is organizational context (why this service exists, who consumes it, what the business constraints are) — information no model has in training data.

Ben Evans' analysis of AI job exposure aligns: roles with high tacit knowledge and cross-functional dependencies are AI-resistant. Developers who write specs, design system boundaries, and verify agent outputs are the new senior engineers. Developers who only type code are competing against $0.01/1K token completion APIs. The career moat is moving from "can you code?" to "can you specify and validate?"
