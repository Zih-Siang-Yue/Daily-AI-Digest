MLX Swift Run Open-Source LLMs at 40–60 tok/s on iPhone with MLX Swift

Apple's MLX framework has matured into a production-ready solution for on-device LLM inference on iOS. Using the MLX Swift LM package, you can now download quantized models (Gemma 4, Qwen 3.5, Phi-4) directly from Hugging Face and run them at 40–60 tokens/second on A18/M4 hardware — without any Python or ONNX conversion pipeline. The key architectural advantage: MLX uses Apple's Unified Memory Architecture (UMA) for zero-copy GPU sharing, meaning no data movement between CPU and GPU. Core ML vs MLX Route Inference by Power State: ANE for Background, GPU for Foreground

The 2026 best practice for iOS AI apps: use Core ML (Neural Engine) for always-on background tasks and MLX (GPU) for interactive foreground generation. Core ML INT8 models use ~240MB RAM and are extremely battery-efficient via the ANE, but lack streaming speed. MLX INT4 models deliver chat-speed generation but consume more power. Build a two-path router: check ProcessInfo.processInfo.thermalState and UIApplication.shared.applicationState — foreground + cool thermal → MLX GPU path; background or elevated thermal → Core ML ANE path. This single architectural split cuts energy consumption 40–60% for apps with persistent AI features.

Foundation Models Use Apple's Foundation Models Framework for Zero-Latency Local NLP

Apple's Foundation Models framework lets you prompt the on-device SLM powering Apple Intelligence in just 3 lines of Swift — no model downloads, no network calls. Use it for text summarization, entity extraction, and classification tasks where privacy and instant response matter. The framework returns native Swift types (not JSON strings), eliminating parsing boilerplate. For iOS apps at Binance, this means user-facing NLP features (transaction categorization, support message triage) can run entirely on-device with zero latency. Reserve MLX/Core ML for heavier tasks like code generation or long-form chat. Apple Docs →
