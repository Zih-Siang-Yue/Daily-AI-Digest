Neural Engine · Scheduling Neural Engine Power Scheduling: Run ML Inference During Idle Time Without Draining Battery

Most iOS developers fire ML inference on-demand (user taps a button → run model). But the power-efficient pattern is background scheduling: use BGProcessingTask combined with ProcessInfo.thermalState checks to run expensive ML inference only when the device is charging, cool, and idle. Architecture: (1) Register a BGProcessingTaskRequest with requiresExternalPower = true. (2) In the handler, check ProcessInfo.processInfo.thermalState — abort if above .fair. (3) Run batch inference (process last 24h of user data, pre-compute recommendations, update personalized models). (4) Cache results in a local SQLite store for instant retrieval when the user opens the app. This pattern enables computationally expensive ML features without any battery impact — the user never notices. For Binance iOS: run overnight portfolio risk analysis, pre-compute personalized market alerts, or update the on-device anomaly detection model (covered June 23) during charging. The Neural Engine draws only 1-3W during inference — comparable to the idle radio, invisible on the battery chart. Combine with MLComputeUnits.cpuAndNeuralEngine to avoid GPU thermal pressure from background tasks. Apple BGTask Docs →
