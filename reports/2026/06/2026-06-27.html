Best of Week · Tip #1 · Vibe Coding
Supervisor-Worker Multi-Agent: The Production-Ready Pattern

This week's most-cited pattern: a supervisor agent that receives a high-level task, breaks it into sub-tasks, spawns specialised worker agents, monitors their outputs, and retries failures — all without human intervention in the loop.

The winning structure:

  • Supervisor holds the task_graph and tracks state across workers
  • Workers are stateless and receive only the context they need (reduces hallucination)
  • Supervisor validates each worker output against a schema before proceeding
  • On failure, supervisor re-queues with a narrower task definition

This pattern was seen in Claude Code hooks, Cursor Background Agents, and Copilot Agent Mode all shipping similar primitives within days of each other — signal that it's becoming the de-facto standard.

Best of Week · Tip #2 · AI Engineering
Structured Tool Use: Always Return Typed Schemas, Never Free Text

The week's most actionable developer pattern: when an LLM calls a tool, never return raw free-form text back to the model. Return a strongly-typed JSON schema with a fixed field set.

Why it matters:

  • Free-text tool results cause the model to "interpret" rather than "act", ballooning token usage
  • Typed schemas allow you to validate upstream, cutting 30–60% of re-try loops in production
  • Use strict: true in OpenAI / Anthropic tool definitions — forces the model into schema-compliant outputs

Combined with Contextual Retrieval (also this week), this makes RAG pipelines dramatically more reliable: retrieve typed chunks → inject as structured tool results → model acts, not interprets.
