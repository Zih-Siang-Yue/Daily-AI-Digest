CoreML · Personalization CoreML Adaptive Models — Personalize On-Device ML Without Retraining

CoreML's MLUpdateTask API enables on-device model personalization without server round-trips. The pattern: ship a base .mlmodel trained on generic data, then fine-tune it locally using the user's behavior. Architecture: (1) Create an updatable model in Create ML by enabling the "Is Updatable" flag on the final layers. (2) Collect user interaction data locally (taps, swipes, time-on-screen) into a MLBatchProvider. (3) Schedule updates via MLUpdateTask during charging (combine with BGProcessingTask). Vision · Pipeline Vision + CoreML Pipeline — Chain Multiple Models for Complex iOS Features

Most iOS developers run a single CoreML model in isolation. The power move: chain multiple models using Vision's VNSequenceRequestHandler to build multi-step AI pipelines that run entirely on the Neural Engine. Pattern: (1) First model: object detection (find regions of interest). (2) Second model: classification (identify what's in each region). (3) Third model: action prediction (determine what to do with the classification). Example for a trading app: detect chart patterns → classify pattern type (head-and-shoulders, double-bottom, etc.) → predict confidence score for the pattern. All three models execute in a single Neural Engine pass using VNImageRequestHandler with shared pixel buffer — zero memory copies between stages. The pipeline runs at 60fps on recent iPhones, enabling real-time chart analysis overlays. Key API: create a VNCoreMLRequest for each model, add all to a single VNImageRequestHandler.perform([request1, request2, request3]) call. Vision automatically optimizes the execution graph.
