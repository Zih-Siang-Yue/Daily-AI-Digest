RAG · iOS Local RAG on iOS: Embed a Vector Store in Your App Bundle

Retrieval-Augmented Generation doesn't require a cloud backend. The pattern for on-device RAG in iOS: (1) Pre-compute embeddings for your knowledge base using NLEmbedding (Apple's Natural Language framework) — supports sentence-level embeddings natively. (2) Store vectors in a lightweight SQLite database with the sqlite-vss extension (vector similarity search, ~200KB overhead). (3) At query time, embed the user's question with the same NLEmbedding.sentenceEmbedding(for:) call, run a cosine-similarity search against your local store, and feed the top-K results as context to your on-device LLM (MLX Swift or a remote API). The architecture gives you RAG with zero network latency for the retrieval step — critical for offline-first experiences. For Binance iOS: embed your FAQ corpus, help documentation, and common error messages as vectors in the app bundle. When users ask questions in support chat, retrieve relevant docs locally in <5ms and generate answers with context — even without internet. Bundle size overhead: ~15MB for 10,000 document chunks. Tool Use · MCP Designing Agent-Friendly APIs: The Tool Schema That Gets 95% First-Try Success

If you're exposing your iOS app's capabilities to AI agents via MCP (Model Context Protocol) or function calling, the schema design determines whether agents succeed on the first attempt or loop endlessly. The proven pattern: (1) Every tool returns a typed JSON envelope: {"status": "ok"|"error", "data": {...}, "suggestion": "..."}. The suggestion field on errors tells the agent exactly what to try next. (2) Use enum parameters with exhaustive descriptions instead of free-form strings — "action": {"enum": ["buy","sell","cancel"], "description": "..."}. (3) Include a "context" parameter that accepts the previous tool's output hash — this creates implicit chaining without the agent needing to parse intermediate results. (4) Add a no-op think tool that accepts a reasoning string — agents use this to "show their work" at decision points, reducing hallucination by 30-40%. For Binance iOS MCP servers: expose granular tools like get_portfolio_summary(), check_order_status(id: String), explain_fee(pair: String). Agent Architecture The Supervisor-Worker Pattern: Scale AI Coding Agents Without Losing Quality

The dominant architecture pattern emerging across Claude Code, Codex, and Cursor's background agents: supervisor-worker separation. The supervisor agent plans and validates; worker agents execute and report. Implementation for your own workflows: (1) Supervisor: receives the high-level task, decomposes it into subtasks, assigns each to a worker, and validates results before merging. Uses a high-capability model (Opus/o3) with a focused system prompt. (2) Workers: each handles one atomic subtask (write a function, fix a test, update docs). Uses a fast model (Sonnet/4o-mini) with narrow file-scoped context. (3) Handoff protocol: supervisor passes a structured task object {"files": [...], "objective": "...", "constraints": [...]} — never the entire conversation history. (4) Quality gate: supervisor runs lint + typecheck + test on worker output before accepting. This architecture gives you 3-5x throughput on large refactoring tasks while maintaining code quality. The key insight: the supervisor's job is verification, not generation — it's cheaper to validate than to produce, so use your best model for validation. Agent Patterns →
