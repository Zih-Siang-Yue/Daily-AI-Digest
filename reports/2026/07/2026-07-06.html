Swift · Structured Concurrency TaskGroup + AsyncStream: The Pattern for Parallel AI Inference in Swift

When running multiple ML models in sequence (e.g., OCR → classifier → summarizer), the temptation is serial await calls. But Swift's TaskGroup lets you parallelize independent inference steps while AsyncStream feeds results to the UI incrementally. The pattern: (1) Create a withTaskGroup(of: InferenceResult.self) that spawns parallel model calls for independent inputs. (2) As each task completes, yield results into an AsyncStream that the SwiftUI view observes via for await. (3) Use Task.priority = .userInitiated for latency-sensitive inference and .utility for background pre-computation. (4) Cancel the group on view disappearance via .task modifier's automatic cancellation. For Binance iOS: when a user opens the portfolio screen, run price prediction, risk scoring, and news relevance classification in parallel across 3 models — display results as they arrive rather than waiting for all three. Total latency drops from 240ms (serial) to ~90ms (parallel). TaskGroup Docs →
