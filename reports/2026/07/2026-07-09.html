Security · HalluSquatting Defend Against HalluSquatting: AI Coding Assistants Can Install Malware

A newly identified attack vector called HalluSquatting exploits AI coding assistants' tendency to "hallucinate" plausible but non-existent package names. Attackers register these fake packages on npm/PyPI with malicious payloads — and when an AI suggests npm install totally-real-utils, you get compromised. The defense pattern for your Cursor/Claude Code/Copilot workflow: (1) Add to your project rules file (.cursor/rules or CLAUDE.md): "Before suggesting any package install, verify the package exists on the registry and has >1000 weekly downloads." (2) Use npm info <package> or pip show <package> as a verification gate before accepting AI-suggested dependencies. (3) Pin exact versions in lockfiles — never accept "latest" from AI suggestions. (4) For iOS/Swift: use Swift Package Manager's URL-based resolution (GitHub repos only) which is inherently resistant since package names map to verified URLs. The broader lesson: treat AI-suggested dependencies with the same skepticism as user input — validate before trusting. Cursor Rules Docs →
