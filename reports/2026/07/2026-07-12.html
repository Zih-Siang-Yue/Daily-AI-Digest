Trending Event-Driven Architecture Goes Mainstream — The AI Agent Communication Layer

As AI agents proliferate (Claude Code spawning sub-agents, Cursor's 8 parallel agents, Copilot's issue-to-PR automation), the industry is converging on Event-Driven Architecture (EDA) as the communication backbone for multi-agent systems. The pattern: agents communicate via typed events rather than direct function calls or shared state. This enables: (1) Loose coupling — agents can be added/removed without breaking others. (2) Auditability — every agent action is an event in an immutable log (satisfying the regulatory requirements from FCA/central bank warnings). (3) Replay and debugging — reproduce any agent behavior by replaying its input events. (4) Rate limiting at the bus level — prevent runaway agents without modifying agent code. For iOS: if you're building an agent-powered feature (AI assistant → API calls → notifications → trade execution), model it as an event stream rather than imperative method calls. Use Swift's AsyncStream + Combine for the event bus. This architecture scales from 1 agent to 100 without refactoring. Martin Fowler: Event-Driven →
