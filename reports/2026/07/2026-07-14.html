Claude Code · Hooks Claude Code Hooks: Automate Pre/Post Actions Without Manual Approval

Claude Code's Hooks system lets you define automated actions that fire before or after agent operations — replacing manual approval fatigue with programmatic guardrails. The setup: create a .claude/hooks.json in your project root with three trigger points: pre-execute (runs before any shell command), post-edit (runs after file changes), and pre-commit (runs before git operations). Practical patterns: (1) Auto-lint on edit: {"trigger": "post-edit", "glob": "**/*.swift", "run": "swiftlint lint --path ${file}"} — every AI-edited Swift file gets linted immediately. (2) Security gate: {"trigger": "pre-execute", "deny_if": "contains(command, 'curl') && !contains(command, 'localhost')"} — blocks outbound network calls from AI-generated scripts. (3) Test runner: {"trigger": "post-edit", "glob": "**/Tests/**", "run": "swift test --filter ${module}"} — automatically runs affected tests after AI edits test files. The key insight: hooks convert "approve/deny" friction into "pass/fail" automation. Your security posture improves AND your flow state is uninterrupted. Hooks Documentation →

Trending Intent-Based Architecture: Declaring WHAT, Not HOW, in the Age of AI

A new architectural philosophy is gaining momentum: Intent-Based Architecture — where developers declare desired system behaviors as high-level intents, and AI agents (or policy engines) determine the implementation. The pattern borrows from Kubernetes' declarative model but applies it to application logic: instead of writing imperative code for "retry 3 times with exponential backoff," you declare intent: resilient_call(max_latency: 2s, availability: 99.9%) and the system determines the optimal strategy. For iOS developers: this maps directly to SwiftUI's declarative paradigm — you already declare UI intent (what it should look like) rather than imperative drawing commands. Intent-Based Architecture extends this to networking, state management, and business logic. The practical benefit: intent declarations are stable across framework migrations, while implementations change. Your AI coding agent can rewrite HOW without changing WHAT — making large-scale refactoring safer and cheaper. Martin Fowler: Declarative Architecture →
