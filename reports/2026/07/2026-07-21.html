Claude Code · Hooks Claude Code Hooks: Automate Pre/Post Actions for Every Agent Command

Claude Code's hooks system lets you define automated actions that fire before or after specific agent behaviors — turning your AI assistant into a disciplined workflow engine. The pattern: create a .claude/hooks.json file in your project root with trigger-action pairs. Practical hooks for iOS developers: (1) Pre-edit hook: {"trigger": "before_file_edit", "action": "git stash push -m 'pre-ai-edit'"} — automatically stashes your work before Claude modifies files, creating an instant rollback point. (2) Post-edit hook: {"trigger": "after_file_edit", "pattern": "*.swift", "action": "swiftlint lint --path ${file}"} — every Swift file Claude touches gets linted automatically. If linting fails, Claude sees the error and self-corrects. (3) Pre-commit hook: {"trigger": "before_commit", "action": "swift build 2>&1"} — prevents Claude from committing code that doesn't compile. (4) Session-start hook: {"trigger": "session_start", "action": "git fetch origin main && git log --oneline -5 origin/main"} — gives Claude awareness of recent team changes every session. The key insight: hooks transform Claude from a "do what I say" tool into a "do what I say within these guardrails" system. Your quality standards are enforced automatically, not by memory. Trending Local-First Architecture: Offline-Capable Apps as the New Default

The local-first architecture movement has reached critical mass in 2026: storing application data on-device as the primary source of truth, with cloud sync as a secondary background process. The tooling maturation is the inflection point — CRDTs (Conflict-Free Replicated Data Types) and efficient sync engines now handle the hard problems (merge conflicts, offline edits, multi-device consistency) that previously required custom solutions. Benefits that align with mobile development: (1) Sub-millisecond UI responsiveness — no network round-trip for data access. (2) Full offline functionality — apps work identically without connectivity. (3) Privacy by default — user data lives on their device, reducing compliance burden. (4) Reduced server costs — cloud becomes a sync relay, not a primary datastore. For iOS at Binance: apply local-first to your watchlist, price alerts, and portfolio views. Users set up configurations locally, the app syncs in background, and everything works on airplanes or in areas with spotty reception. SwiftData + CloudKit already provides a local-first foundation; layer CRDTs (via packages like swift-crdt) for multi-device conflict resolution. The paradigm shift: design for offline first, then add sync — not the reverse. Local-First Web →
